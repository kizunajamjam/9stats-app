# 9stats

9人制バレーボールの試合で、点数や選手ごとの記録（サーブ・レシーブ・アタックなど）をスマホでつけられるアプリです。特別な準備やインストールは不要で、ブラウザでURLを開くだけで使えます。

使えるURL: https://kizunajamjam.github.io/9stats-app/

アプリの中にも「使い方」という画面があり（ホーム画面から開けます）、ここに書いてあるのと同じ説明を読めます。

## 画面は4つ

- **ホーム画面**：「記録」「設定」「実績」「使い方」を選ぶ入り口です。
- **記録画面**：試合中に点数や選手の記録をつけるメインの画面です。スマホを縦に持っていても、自動で横向きの表示になります。
- **設定画面**：試合の情報（日時・会場・チーム名）、セットの点数の決め方、選手の登録（最大20人）を行います。
- **実績画面**：これまでに保存した試合の一覧・詳しい記録・ファイル出力・削除ができます。

## 記録画面の使い方

### 上部のボタン

| ボタン | 何をするか |
|---|---|
| ホーム | ホーム画面に戻ります |
| 自 / 相 の − + | 点数を手動で増やしたり減らしたりします。設定した点数・差に届くと「このセットを終わりにしますか？」という確認が出ます |
| 丸いボタン（①②③…） | 押すとそのセットの記録を見られます（表示の切り替えだけです）。自分のチームが取ったセットは青、相手が取ったセットはオレンジ、まだ終わっていないセットは色がつきません。メモがあると赤い点が付きます |
| 1st / 2nd | サーブが何本目かを表します。1本目のサーブが失敗しても点は取られませんが、2本目も失敗すると相手に点が入ります |
| メモ | 今見ているセットのメモを入力・編集します。この画面だけ縦向きで表示されます（キーボードの向きを合わせるため） |
| 戻す | 直前の操作を取り消します（さかのぼれるのは15回前までです） |
| R | 点数と記録だけを0に戻します。登録した選手や背番号、試合の情報は消えずに残ります |
| 終了 | 試合が終わったときに押します。これまでの記録を「実績」に保存してから、点数と記録を0に戻してホーム画面に戻ります |

### 選手一覧

- **背番号のボタン**：押すとベンチにいる選手と交代できます（出ている選手を消すことはできません。交代だけができます）。
- **選手名**：記録画面では名前を変えられません。名前を変えたいときは設定画面で行います。
- **記録ボタン（12個並んでいます）**：
  1. サーブ：P＝決まった ／ M＝失敗した
  2. レシーブ：A＝とても良い ／ B＝良い ／ C＝普通 ／ D＝失敗
  3. アタック：P＝決まった ／ M＝失敗した
  4. ブロック：P＝決まった ／ M＝失敗した
  5. その他：P＝得点 ／ M＝失敗

終わったセットを見ているときは、数字を確認できるだけでボタンを押すことはできません。

## 設定画面の使い方

- **試合の情報**：日時・会場・自分のチーム名・対戦相手の名前を入力できます。
- **セットの点数 / 差**：1セットの終わり方を決めます。「差」を0にすると、決めた点数に届いた時点でそのセットがすぐ終わります。
- **セットの先取数**：ここで決めた数のセットを先に取ったチームが出ると「試合が終わりました」という案内が出ます（記録の保存は「終了」ボタンを自分で押してください）。
- **選手の登録**：最大20人まで登録できます。同じ背番号は2人以上登録できません。登録した選手を消すことはできず、出ている選手との交代だけができます。

## 実績画面の使い方

- 記録画面で「終了」を押すと、その試合がここに表示されます。それぞれに最後に保存した日時も表示されます。
- 試合をタップすると、その試合全体の選手ごとの記録と、表計算ソフトで開けるファイルとして保存できるボタンが出てきます。「合計」「セット1」「セット2」…のボタンで、試合全体とセットごとの記録を切り替えられます。セットにメモがあればそこに表示されます。
- 「削除」ボタンを押すとその試合の記録が消えます（消した記録は元に戻せません）。

## 記録が消えないようにするには

- 記録中の試合は、ボタンを押すたびに自動でこの端末に保存されます。画面を閉じたり、ページを更新しても記録は残ります。
- ただし、スマホやブラウザのデータを消す操作（キャッシュの削除など）をすると記録も一緒に消えてしまいます。大事な記録は実績画面からファイルとして保存しておくことをおすすめします。
- アプリが新しくなったときに記録の形式が変わることがあり、その場合は試合中のデータが新しい試合として最初からになることがあります（すでに保存し終えた実績のデータは影響を受けません）。

### このアプリを動かしている人向けの補足

特別な仕組みは使わず、見たままのファイル（`index.html` / `style.css` / `app.js`）だけで動くシンプルな作りです。GitHubの「Pages」という無料の公開機能を使って、上記のURLでだれでも見られるようにしています。中身を直して保存（push）すれば、数十秒〜数分でそのまま公開されます。
