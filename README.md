# 9stats

9人制バレーボール専用のスタッツ記録PWA（Progressive Web App）です。ビルド不要のプレーンなHTML/CSS/JavaScriptで構成されており、スマートフォンのブラウザでそのまま動作します。

公開URL: https://kizunajamjam.github.io/9stats-app/

アプリ内にも「使い方」画面（ホーム画面から遷移）があり、この説明書と同じ内容を確認できます。

## 画面構成

アプリは4つの画面で構成されます（画面切り替えはJavaScriptが行い、ページ遷移はありません）。

- **ホーム画面**：「記録」「設定」「実績」「使い方」への入り口。
- **記録画面**：試合中にスコア・選手スタッツを記録するメイン画面。縦持ちでも自動的に横向き表示に固定されます。
- **設定画面**：試合情報、セットルール、選手登録（最大20名）を管理します。
- **実績画面**：保存済みの試合記録の一覧・詳細表示・CSV出力・削除。

## 記録画面

### ヘッダー

| 項目 | 説明 |
|---|---|
| ホーム | ホーム画面に戻る |
| 自 / 相 の − + | スコアを手動で増減する。セット点数・デュース差の条件を満たすと「セットを終了しますか？」の確認ダイアログが出る |
| セットバッジ（①②③…） | タップでそのセットの記録（選手・スタッツ）を表示する。自チームが取ったセットは**青**、相手が取ったセットは**橙**、進行中のセットは無色。表示中のセットには黒枠がつく |
| 1st / 2nd | サーブの状態表示。1本目のミスは失点にならず2ndサーブへ。2本目のミスで相手に1点入る（9人制バレーの2サーブ制ルール） |
| 戻す | 直前の操作を取り消す（最大15件のUndo履歴） |
| R（リセット） | スコア・スタッツのみ初期化。選手登録・背番号・試合情報は保持される |
| 終了 | 試合を終了する。現在の記録（全セット通算）を実績へ保存し、スコア・スタッツをリセットしてホームへ戻る |

### 選手リスト（9スロット）

- **背番号ボタン**：タップでベンチにいる選手と交代できる。出場メンバーの削除はできない（交代のみ）。
- **選手名**：記録画面では表示専用。変更は設定画面の選手登録から行う。
- **記録ボタン（12列）**：
  1. サーブ：P（得点） / M（失点）
  2. レシーブ：A（優） / B（良） / C（可） / D（不可＝失点）
  3. アタック：P（得点） / M（失点）
  4. ブロック：P（得点） / M（失点）
  5. その他：P（得点） / M（ミス）

過去のセットを表示しているときは数値の確認のみで、ボタンは押せない（閲覧専用）。

## 設定画面

- **試合情報**：日時・会場・自チーム名・対戦相手。
- **セットルール**
  - セット点数：1セットの目標点数（デフォルト21点）
  - デュース差：何点差で決着するか（0にするとデュースなし＝セット点数到達で即終了）
  - 先取セット数：このセット数を先取したチームが出ると試合終了の通知を表示（デフォルト2＝3セットマッチ）。保存・リセットは「終了」ボタンで手動で行う
- **選手登録**：最大20名まで登録可能。背番号は重複登録不可。登録済み選手は削除できず、出場中選手との交代のみ可能（出場中/ベンチの状態が一覧に表示される）。

## 実績画面

- 「終了」ボタンで保存された試合が一覧表示される。各項目に最終更新日時が表示される。
- 項目をタップすると、その試合の選手別スタッツ（全セット通算）の詳細テーブルとCSV出力ボタンが表示される。
- 「削除」ボタンでその試合の記録を消去できる（元に戻せない）。
- 同じ試合中に「終了」を複数回行うことはない想定だが、内部的には1つの試合につき1レコードとして上書き保存される。

## データの保存

- 記録中の試合データはlocalStorage（ブラウザ内ストレージ）に操作ごとに自動保存される。画面を閉じたりリロードしても記録は失われない。
- 実績（試合履歴）も同様にlocalStorageに保存される。**端末・ブラウザのキャッシュを削除すると記録は失われる**ため、重要な記録は実績画面のCSV出力でバックアップすることを推奨。
- アプリの内部データ構造（スキーマ）が変わるバージョンアップを行った場合、保存されていた進行中の試合データは読み込めず新規試合として初期化される（過去に保存済みの実績データへは影響しない）。

## 技術構成・デプロイ

- ビルドステップなしのプレーンなHTML/CSS/JavaScript（`index.html` / `style.css` / `app.js`）。
- PWA対応（`manifest.json`）。ホーム画面に追加してアプリのように起動できる。
- [GitHub Pages](https://kizunajamjam.github.io/9stats-app/) でホスティング。`master`ブランチのルートが公開ソースに設定されており、`git push`するだけで数十秒〜数分で本番に反映される。

## ファイル構成

```
index.html    画面のHTML（ホーム/記録/設定/実績/使い方/交代モーダル）
style.css     スタイル（白基調・横向き固定レイアウト含む）
app.js        アプリのロジック（状態管理・画面遷移・記録処理・保存）
manifest.json PWA設定
icon.png / icon-512.png  アプリアイコン
```
